HOW WEIGHTS WORK
================

RIGHT NOW (no file here):
  The pipeline uses YOLOv8n pretrained on COCO (80 general classes).
  ultralytics downloads it automatically on first run (~6 MB).

  What you get:
    ✅  Accurate bounding boxes around objects
    ✅  ISA tags attached via OCR (FCV-101, PT-745, etc.)
    ❌  Class names will be COCO labels (person, car, truck...)
        not P&ID symbols (gate_valve, pressure_indicator, etc.)

  This is still useful — the graph has correct tags, just wrong class names.

WHEN YOU HAVE FINE-TUNED WEIGHTS:
  1. Copy your weights here:
       cp /path/to/runs/detect/pid/weights/best.pt  weights/best.pt

  2. Run exactly the same way:
       python predict.py diagram.jpg

  The pipeline auto-detects the file and uses it instead of the default.

FINE-TUNING INSTRUCTIONS
=========================

1. Collect P&ID images (aim for 100+ drawings, or at least 50)

2. Label symbols using Roboflow (easiest) or Label Studio:
     https://roboflow.com
     https://labelstud.io
   Export format: YOLOv8  → gives you  images/  labels/  data.yaml

3. Train:
     pip install ultralytics
     yolo train \
       model=yolov8n.pt \
       data=/path/to/data.yaml \
       epochs=100 \
       imgsz=1280 \
       batch=8 \
       project=runs/pid

4. Copy best weights:
     cp runs/pid/trainN/weights/best.pt  weights/best.pt

5. Update class list if needed:
   Open  pid_graph/config.py  → SYMBOL_CLASSES
   Make sure the list matches your data.yaml class names in the same order.

6. Run:
     python predict.py diagram.jpg --conf 0.25
